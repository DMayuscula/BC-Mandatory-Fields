# BC-Mandatory-Fields
AL development for the Mandatory fields extension.

## How to use the extension: 

First we need to enter the MDF Table Config List, in here you have the following options: 

<img width="1190" height="54" alt="imagen" src="https://github.com/user-attachments/assets/c07c1f2f-8058-4ffd-a4de-c14ae82dc2b4" />

Load Fields From Table: 

Here is your first step, in here you can select one table from all the tables available in the system, once you select the table where you want to configure the mandatory fields, you select the Create Fields option in the corresponding page: 

<img width="1105" height="545" alt="imagen" src="https://github.com/user-attachments/assets/c115fddb-6cd7-42ea-9bf7-ee998ef62221" />

We are going to use the Customer table for this example, we clic on the Create Fields option and the following confirmation message shows up: 

<img width="507" height="180" alt="imagen" src="https://github.com/user-attachments/assets/0227d623-6deb-4e72-a769-c40bf1f4781d" />

Now we have a new record to use the following Mandatory Fields option, when we click in this action well get the full list of all the fields (custom included) from the table we selected, in here we can select which items we want to have as mandatory fields: 

<img width="1019" height="712" alt="imagen" src="https://github.com/user-attachments/assets/0ee50fb1-7c12-4274-80a5-09458dc0c5cb" />

With this we now have the following mandatory fields: 
Name
Address
City
Phone No.
Customer Posting Group.

Also we can edit the MDF Table Config List to activiate the Block on Validation option, this activates the function to block the new record if the mandatory fields are not filled.

<img width="1021" height="186" alt="imagen" src="https://github.com/user-attachments/assets/be71f6e0-6716-4ca9-b42b-b617e982cdab" />

In case we need to reset and reload the fields of the selected table, we can use the Reset & Reload Fields option.
